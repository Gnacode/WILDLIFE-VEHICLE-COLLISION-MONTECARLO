05 Methods summary

All results were generated by a single Python simulator, wvc_simulator.py, using a behaviorally-realistic discrete-time Monte Carlo framework on a 1 km test corridor.

System architecture

Alternating-side Doppler radar nodes at 15 m spacing (50 sensors/km baseline, 67/km at 15 m), three-axis magnetometer sites every 200 m, and dynamic message signs (DMS) every 250 m. Sensors communicate over an LVDS+LoRa hybrid backbone; an awareness signal propagates between neighboring radars to boost detection sensitivity by a factor of 1.8× during 30 s windows around recent detections.

Vehicle dynamics

Treiber–Hennecke–Helbing Intelligent Driver Model (IDM) with desired gap s₀ = 5 m, time headway T = 1.5 s, comfortable acceleration a = 2.5 m/s², comfortable deceleration b = 4.0 m/s², emergency braking 9.0 m/s². Driver reaction delay 1.5 s.

Animal behaviour

Six-state Markov model: Foraging → Approaching → Hesitating → {Crossing | Frozen | Fleeing} → Gone. Behavioural transitions at the road edge are gated by vehicle threat. Let θ = min(1, vvehicle/vcruise) be the speed-derived threat factor on a [0, 1] scale. Two branches:

  • Clear branch (no vehicle within 80 m, or θ ≤ 0.3): p(cross) = 0.80, p(flee) = 0.20.
  • Threat branch (vehicle within 80 m AND θ > 0.3): p(frozen) = 0.10·θ, p(flee) = 0.30 + 0.20·θ, p(cross) = 0.70 − 0.30·θ.

Mid-crossing freeze (animal already on the road): rate p(freeze) = 0.15·θ·Δt per time step when a vehicle approaches within 40 m. The activation threshold θ > 0.3 is calibrated against the caution-speed setpoint of 30 km/h: a DMS-alerted vehicle decelerates below this threshold and is no longer perceived as a threat, redirecting probability mass from the threat branch (freeze, flee) back toward the clear-branch crossing outcome. This branch-redirection is the principal behavioural mechanism by which driver alerts reduce collisions.

Three operating modes

  • Control — no sensors, no alerts, no caution speed.
  • Detection — sensors active, drivers receive DMS warnings and reduce to caution speed when in any active awareness zone, no neighbor-sensor boost.
  • Aware — Detection mode plus LoRa-propagated awareness boost (1.8× sensitivity on neighboring radars for 30 s after each detection).

Statistical analysis

All p-values from two-sided Welch's t-tests on per-trial outcomes. Significance markers: *** p < 0.001, ** p < 0.01, * p < 0.05, n.s. otherwise. The headline result is the collision rate per road entry, which normalises for differing crossing throughput between modes; the absolute collision count differs by mode mainly through the throughput-vs-rate trade-off discussed in the Headline section. Sweep figures use shaded ±1 SEM bands (SD/√n) around the per-point mean, showing confidence in the mean estimate at each sweep value; trial-level variability is shown explicitly in the headline box plots.

06 Data and reproducibility

All figures and tables are generated directly from the CSVs by this repository's analysis script. Re-running the script on the raw data reproduces every number on this page byte-identical.

Source CSVs

  • wvc_results.csv — headline three-mode dataset
  • wvc_results_spacing.csv — sweep over radar_spacing
  • wvc_results_size.csv — sweep over size_scale
  • wvc_results_detection_rate.csv — sweep over detection_rate_per_sec

Reproducing the figures

git clone <this-repo>
cd <this-repo>
python -m pip install pandas plotly scipy numpy
python analyze_wvc.py --input data --output docs

Reproducing the raw data

The simulator wvc_simulator.py is fully self-contained and uses only standard scientific Python. To regenerate every CSV on this page in a single run:

python regenerate_all_data.py            # all sweeps (~30–60 min)
python regenerate_all_data.py --skip-sweeps   # headline only (~5 min)
